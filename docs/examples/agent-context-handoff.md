# 怎么让 Agent 直接读你的上下文

目标很简单：

别再每次开新对话，都重新解释你最近做了什么。

## 最短路径

把 OpenMy 装好以后，让 Agent 先读这几个稳定入口：

```bash
openmy skill status.get --json
openmy skill context.get --json
openmy skill day.get --date 2026-04-13 --json
```

## 这三步分别解决什么

1. `status.get`：先看系统有没有数据，别一上来就瞎猜。
2. `context.get`：先吃掉跨天上下文，知道最近重点项目和待办。
3. `day.get`：如果你今天刚处理过某一天，再补读这一天的细节。

## 一个实际对话开场

你可以直接对 Agent 说：

> 先读我的 OpenMy 上下文，再继续昨天那个任务。

正常情况下，Agent 应该自己先读上下文，再进入正题，而不是先问你“最近在做什么”。

## 最稳的接法

- 先接 `skill` 入口，不要先依赖旧的 `agent` 兼容入口。
- 先读 JSON，再自己整理成人话，不要把原始 JSON 直接贴给用户。
- 先读 `context.get`，再决定要不要补读 `day.get`。

## 常见坑

### 坑 1：只读 README，不读技能文档

这样最容易半路卡住。

### 坑 2：把 OpenMy 当成录音工具

它不是拿来现场录音的。
它是拿来处理已经落盘的录音和屏幕上下文的。

### 坑 3：没上下文就直接问用户一堆问题

只要 OpenMy 里已经有上下文，Agent 就该先读，再问真正缺的那一点。
