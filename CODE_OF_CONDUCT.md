# Code of Conduct

## Our Commitment

We want OpenMy to be a welcoming, low-drama, high-signal project for contributors, users, and maintainers.
Everyone participating in this repository, its discussions, and related community spaces is expected to act with respect, empathy, and professionalism.

## Expected Behavior

Examples of behavior that help create a positive environment include:

- being respectful of different viewpoints and experience levels
- giving actionable, technically grounded feedback
- assuming good intent while still being clear and direct
- accepting responsibility, apologizing, and learning from mistakes
- focusing criticism on ideas, code, and behavior rather than people

## Unacceptable Behavior

Examples of unacceptable behavior include:

- harassment, stalking, or intimidation
- hateful, discriminatory, or sexualized language or imagery
- personal attacks, insults, dogpiling, or sustained hostility
- publishing private information without permission
- deliberately derailing discussions or spamming project spaces

## Scope

This Code of Conduct applies within all project spaces, including:

- GitHub issues, pull requests, and discussions
- documentation, review threads, and support channels
- any public or private interaction where someone is representing the project

## Reporting

If you experience or witness unacceptable behavior, contact the maintainers privately.
If a private maintainer email is not yet published, use GitHub's private maintainer contact paths (for example, direct maintainer contact or a private security advisory) instead of opening a public issue.

Please include:

- what happened
- where it happened
- when it happened
- any screenshots, logs, or links that help establish context

## Enforcement

Maintainers may remove, edit, or reject comments, commits, code, issues, and other contributions that violate this Code of Conduct.
They may also temporarily or permanently ban contributors from project spaces for behavior they determine to be inappropriate, threatening, abusive, or harmful.

## Attribution

This policy is adapted for OpenMy from the Contributor Covenant spirit and simplified to fit this repository's current operating model.
